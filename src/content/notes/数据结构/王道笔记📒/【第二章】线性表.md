## 2.1 线性表的定义和基本操作

### 2.1.1 线性表的定义

定义：具有**相同数据类型**的 $n$ 个数据元素的**有限序列**。
- $n$ 为表长，$n\geq0$。$n=0$ 时为空表。
- 用 $L$ 表示为：
	- $L=(a_{1},a_{2},\dots,a_{i},a_{i+1},\dots,a_{n})$
	- $a_{1}$：第一个数据元素，也叫**表头元素**。
	- $a_{n}$：最后一个数据元素，也叫**表尾元素**。
- 逻辑特性：
	- 除第一个元素外，每个元素有且仅有**一个直接前驱**。
	- 除最后一个元素外，每个元素有且仅有**一个直接后继**。
- 特点：
	- 表中元素的**个数有限**。
	- 表中元素具有逻辑上顺序性，表中**元素有先后次序**。
	- 表中都是数据元素，每个**元素都是单个元素**。
	- 表中**元素的数据类型都相同**，也就是说每个元素所占据的存储空间相同。
	- 表中元素具有抽象性，**只讨论元素之间的逻辑关系**，不关心元素内容。

> 注意区分**顺序表**和**线性表**，线性表是一种高度抽象的**逻辑结构**，而顺序表是**存储结构**。顺序表和链表在同一层次。
### 2.1.2 线性表的基本操作

基本操作：指一个数据结构**最核心、最基本**的操作。
其他复杂操作：可调用基本操作来实现。

线性表的主要操作有：
- `InitList(&L)`：初始化表。
	- 构造一个空线性表。
- `Length(L)`：求表长。
	- 返回线性表的长度 $L$，即表中的数据个数。
- `LocateElem(L, e)`：按值查找操作。
	- 在表 $L$ 中查找具有给定关键字值的元素。
- `GetElem(L, i)`：按位查找操作。
	- 按位查找操作。获取表 $L$ 中第 $i$ 个位置的元素的值。
- `ListInsert(&L, i, e)`：插入操作。
	- 在表 $L$ 中的第 $i$ 个位置上插入指定元素 $e$。
- `ListDelete(&L, i, &e)`：删除操作。
	- 删除表 $L$ 中第 $i$ 个位置的元素，并用 $e$ 返回删除元素的值。
- `PrintList(L)`：输出操作。
	- 按前后顺序输出线性表 $L$ 中的所有元素的值。
- `Empty(L)`：判空操作。
	- 若 $L$ 为空表返回 true，否则返回 false。
- `DestroyList(&L)`：销毁操作。
	- 销毁线性表，释放线性表 $L$ 所占用的内存空间。

> 1️⃣基本操作的实现取决于存储结构，存储结构不同算法实现也不同。2️⃣ `&` 为 C++中的引用调用。

## 2.2 线性表的顺序表示

> 从字面上就能理解，顺序表是有顺序的线性表，约束条件更强。

顺序表：线性表的**顺序存储**。

特点：
- 用**地址连续的存储**单元依次存储线性表中的数据元素。
- **逻辑相邻**的两个元素在**物理位置上也相邻**。
- 物理存储位置与位序成正比，故可以实现随机存取。

优点：
- 可实现**随机访问**，通过首地址+元素序号 $O(1)$ 时间内找到指定元素。
- 存储密度高，每个节点只能存储数据元素。


> [!NOTE]- 理解
> 为什么链表不具备随机访问呢？原因在于**顺序表的顺序高度确定**，这种强大的起家条件使得顺序表具备了卓越的查询能力。



缺点：
- 插入和删除耗时，可能需要移动大量元素。
- 需要连续空间才能存储，不够灵活。

> 随机存取就是不需要遍历线性表，直接给出一个位序就可以直接存入。

位序：称 $i$ 为元素 $a_{i}$ 在顺序表中的位序。

> 顺序表中元素的位序从 `1` 开始，而数组的下标是从 `0` 开始的。


> [!NOTE]- 数组
> 线性表强调线性，顺序表强调顺序。**数组**是高级语言中的概念，并非数据结构中的抽象概念，比如 Python 中就只有列表而没有数组。如果说线性表以及它下面的项目是一种思想，则**数组**则更多的是一种实际实现（代码层面对顺序表的实现），而非一种思想。
> > [!Error]
> > 这里有一个误区，可能会认为逻辑上数组和顺序表示一样的。实则不然，其他几种数据结构都能用数组来进行实现，比如链表、树🌲、队列、栈都能找到数组的对应实现，浙江的同学可以回忆一下，咱们高中时候学的栈、链表不都是用 Python 中的列表（就是数组）来实现的吗？

> [!NOTE]-  存取方式
> **存取方式**指的是 `写/读`，因此顺序表的存取方式是**随机存取**。可以在任何一个随机的位置用 O(1)复杂度读取数据，写入就是：`arr[2]=4` 这种数组的赋值语句。
> 有些同学（比如我）可能一开始会困惑**插入数据**不也是一种写入吗？实际上不是的，**插入操作不能被理解为存取操作中的“存”**。

## 2.3 线性表的链式表示

### 2.3.1 单链表的定义

- **定义**：线性表的链式存储也称*单链表*。
- **优点：**
	- 解决了顺序表需要大量连续存储单元的缺点。
- **缺点**
	- 不能直接找到表中的某个特定结点，寻找特定结点时需要从表头开始遍历（没有索引这个概念了）。
	- 附加的指针域浪费空间。
	- 每个链表节点除了存放元素自身的信息外，还需要存放一个指向其后继的*指针*（这里我们可以回忆一下顺序表的元素是只能存放单个元素的，有所不同）。


> [!NOTE] 头结点与头指针
> 用*头指针 L（或 head）* 来标识一个单链表，指出链表的地址。
> 头指针为 NULL 时表示一个空表。
> - 带表头结点：头指针 L 指向头结点，头结点可以携带一些链表的信息（如链表长度）。
> - 不带表头结点：头指针 L 指向第一个数据结点。
> - 表尾结点：指针域为 NULL。

> [!QUESTION] 为什么要引入头结点？
> 引入头结点主要是为了解决空链表的问题。引入头结点后空链表的头指针可以指向头指针，这在逻辑上就和非空链表完全一致了（实现了大一统），在处理上也得到了统一。




单链表的定义：

```C
typedef struct LNode {    // 定义了结点的数据类型
    ElemType data;        // 数据域
    struct LNode *next;   // 指针域
}LNode, *LinkList;
```


### 2.3.2 单链表上基本操作的实现

> 从这里开始均为 WD 书上代码的抄写了，难免会有打错，目的主要是为了通过输出带动自己过一遍。

1. 单链表的初始化

这是一个带头结点单链表的初始化，需要创建一个头结点，并让头指针指向头结点，头结点的 next 域初始化为 NULL。

```C
bool InitList(LinkList &L) {            // 带头结点的单链表的初始化
    L=(LNode*)malloc(sizeof(LNode));    // 创建头结点
    L->next=NULL;                       // 头结点之后暂时没有元素结点，此时头结点就是表尾结点
    return true;
}
```

不带头结点的单链表在初始化时，只需要将头指针 L 初始化为 NULL 就可以。

```C
bool InitList(LinkList &L) {
    L=NULL;
    return true;
}
```

> 这里的 `LinkList` 就是头指针。
> 书上 P 31 的 `注意` 部分就是一些简单的 C 语言指针语法，这里省略。


2. 求表长操作

顾名思义，遍历链表求出表的长度。

```c
int Length(LinkList L) {
    int len=0;
    LNode *p=L;
    while(p->next!=NULL) {
        p=p->next;
        len++;
    }
    return len;
    // 这段代码简单，不过多说明（lazy）
}
```

3. 按序号查找结点

- 按照序号搜索结点并返回该结点的指针；
- 如果 `i` 大于表长，则返回 NULL。

```c
LNode *GetElem(LinkList L,int i) {
    LNode *p=L;
    int j=0;
    while(p!=NULL&&j<i) {
        p=p->next;
        j++;
    }
    return p;
}
```

4. 按值查找表结点

- 从第一个结点开始，从前往后以此比较结点的数据域，若某结点的 data 域等于给定值 e，则返回该结点的指针；
- 若整个单链表中没有这样的结点，则返回 NULL。

```c
LNode *LocateElem(LinkList L,ElemType e) {
    LNode *p=L->next;
    while(p!=NULL&&p->data!=e) p=p->next;    // 从第一个结点开始查找数据域为e的结点
    return p;                                // 找到后返回指针否则返回NULL。
}
```

5. 插入结点操作

- 将值为 x 的新结点插入到单链表的第 `i` 个位置。
- 先检查插入位置的合法性，然后找到待插入位置的前驱，即第 `i-1` 个结点，再在其后插入。

```c
bool ListInsert(LinkList &L,int i,ElemType e) {
    LNode *p=L;                    //指针p指向当前扫描到的结点
    int j=0;                       //记录当前结点的位序，头结点是第0个结点
    while (p!=NULL&&j<i-1) {       //循环找到第i-1个结点
        p=p->next;
        j++;
    }
    if (p==NULL) return false;      // i值不合法
    LNode *s=(LNode*)malloc(sizeof(LNode));
    s->data=e;
    s->next=p->next;
    p->next=s;
    return true;
}
```


> [!NOTE] 后插法
> 上面👆都是前插法，即插到目标结点的前面。前插法中通过找到目标结点的前一个结点来实现。而**后插法**直接找到目标结点并先插到其后面，然后通过交换目标结点和新插入的结点来实现，最终效果和前插法是一样的。
> 
> ```c
> s->next=p->next;
> p->next=s;        // 前两句和前插法一样
> temp=p->data;     // 交换数据
> p->data=s->data;
> s->data=temp;
> ```

6. 删除结点操作

- 将单链表的第 `i` 个结点删除。
- 先检查删除位置合法性。
- 然后查找 `i-1` 个结点，也就是被删结点的前驱。
- 再删除第 `i` 个结点。

> 都很好理解，不过多赘述。

```c
bool ListDelete(LinkList &L, int i,ElemType &e) {
    LNode *p=L;
    int j=0;
    while (p->next!=NULL&&j<i-1) {
        p=p->next;
        j++;
    }
    if (p->next==NULL||j>i-1) return false;    // j>i-1是为了防御i<=0时的越界情况
    LNode *q=p->next;    // 此时p是待删除结点的前驱，q是待删除结点
    e=q->data;
    p->next=q->next;
    free(q);
    return true;
}
```


> [!NOTE] 扩展（思想后插法）
> 该方法类似于头插法，找到结点后在后部插入。
> 
> ```c
> q=p->next;                // 此时已经用p指向待删除结点，q指向了待删结点后继
> p->data=p->next->data;    // 覆盖待删结点数据域
> p->next=q->next;          // 断开链接
> free(q);                  // 释放
> ```


7. 采用头插法建立单链表

> 使用头插法创建，输入的顺序和链表中结点的顺序相反。插入时头指针保持指向头结点，将新来的元素插到首个数据元素前面。所以我们可以这样去记忆头插法：**利用头指针在头部插入元素，就叫做头插法**。

```C
LinkList List_HeadInsert(LinkList &L) {
    LNode *s; int x;
    L = (LNode*)malloc(sizeof(LNode));        // 创建头结点
    L->next=NULL;                             // 空链表
    scanf("%d", &x);
    while(x!=9999) {                          // 输入9999结束程序
        s=(LNode*)malloc(sizeof(LNode));      // 分配结点
        s->data=x;                            
        s->next=L->next;                      // 新结点链接到首个数据结点
        L->next=s;                            // 头结点指向的一个数据元素
        scanf("%d", &x);
    }
    return L;
}
```


8. 采用尾插法建立单链表

> 为了保证结点次序与输入时一致可采用尾插法。记忆方式：**利用尾指针在尾部插入新元素，就是尾插法**。

```c
LinkList List_TailInsert(LinkList &L) {
    int x;
    L=(LNode*)malloc(sizeof(LNode));      // 头结点
    LNode *s,*r=L;                        // r为尾指针，s用于指向新分配的结点
    scanf("%d", &x);
    while (x!=9999) {
        s=(LNode *)malloc(sizeof(LNode)); // 创建新结点
        s->data=x;                        
        r->next=s;                        // 尾结点链接到新结点
        r=s;                              // 更新尾结点
        scanf("%d", &x);
    }
    r->next=NULL;                         // 尾结点指向NULL表示尾部
    return L;
}
```


### 2.3.3 双链表

单链表只能往后，不能往前，为了克服，人类设计出了双链表。

1. 双链表的插入操作
```c
s->next=p->next;    // 这一步必须在这里的第四步前面，因为需要使用p->next
p->next->prior=s;
s->prior=p;
p->next=s;
```

2. 双链表的删除操作
```c
p->next=q->next;
q->next->prior=p;
free(q);
```

### 2.3.4 循环链表

1. 循环单链表

- 循环单链表的表中最后一个结点指向的不是 NULL 而是头结点，单链表自然成环了。
- 单链表的判空方式有所不同，当单链表的尾指针指向头结点时表示链表为空（注意区分头结点和第一个数据结点的概念）。
- 单链表只能从表头开始遍历，才能遍历完整张链表。而循环链表从任意位置开始都能够遍历完整张链表。
- 有时对循环单链表仅设尾指针而不设头指针。原因是单链表只能单向遍历，如果设置头指针想要遍历到尾部并插入需要 O(n)复杂度，而设置了尾指针想要遍历到头部只需要往前走一步(`r->next`)即可 O(1)，同时也能很方便地插入到尾部。


1. 循环双链表

和循环单链表类似。

- 链表头指针的 `prior` 还要指向尾结点。
- 循环双链表为空表时，头结点 `prior` 域和 `next` 域都等于 L。

2. 静态链表

- 用数组来描述线性表的链式存储结构，唯一的区别时不需要指针了，将数组索引（按照王道书上说叫做数组的相对地址结构，数组下标）当作地址来用。
- 和顺序表一样，静态链表也需要预先分配一块连续的内存空间。
- 以 `next==-1` 作为链表结束的标志。

> 我个人非常喜欢静态链表这种优雅的写法，相对于计算机分配的地址而言，我认为数组相对索引对程序员来说更加可控。

静态列表的结构类型描述：
```c
#define MaxSize 50    // 静态链表的最大长度
typedef struct {
    ElemType data;    // 存储的数据元素
    int next;         // 下一个元素的数组下标
} SLinkList[MaxSize];
```

### 2.3.6 顺序表和链表的比较

1. 存取方式（读/写方式）
- **顺序表**既可以*顺序存取*，也可以*随机存取*。
- **链表**只能从表头开始依次*顺序存取*。

2. 逻辑结构与物理结构
- 采用**顺序存储**：逻辑上相邻的元素，对应的物理存储位置也相邻。
- 采用**链式存储**：逻辑上相邻的元素，物理存储位置不一定相邻。


3. 查找、插入和删除操作
- 对于**按值**查找：
	- **顺序表**无序时，两者均为 `O(n)`；
	- **顺序表**有序时，采用折半查找时间复杂度为 `O(log2n)`；
- 对于**按序号**查找：
	- **顺序表**支持随机访问，`O(1)`；
	- **链表**的平均复杂度为 `O(n)`；
- 插入、删除操作：
	- **顺序表**平均要移动半个表长元素。
	- **链表**只需修改相关结点的指针域。

4.空间分配
- 顺序存储：
	- **静态存储分配**：一旦存储空间满就不能扩充，再加入新元素会导致内存溢出，因此需要预先分配足够的存储空间。分配的过大会导致后部大量闲置；分配过小又会造成溢出。
	- **动态存储分配**：虽然存储空间可以扩充，但需要移动大量元素，导致操作效率降低，如果内存中没有更大块的连续存储空间，则会导致分配失败。
- 链表：
	- 只要由空间就能分配，缺点是需要浪费数据域存放指针，存储密度低。


> [!NOTE] 💭在实际中怎样选取存储结构呢？
> 
> 1. 基于存储的考虑
>     - 难以估计长度或规模时，***不宜采用顺序表***；
>     - **链表**不用实现估计存储规模，但链表存储密度低。
>     - **链式存储结构**的存储密度小于1。
> 2. 基于运算的考虑
>     - **顺序表**按*序号*访问 $a_{i}$ 显然快得多。若经常做的运算是按序号访问数据元素，则显然顺序表优于链表。
>     - 当*数据元素信息量较大*且*表较长*时，进行插入、删除，显然链表更好。
> 3. 基于环境的考虑
>     - **顺序表**容易实现，任何高级语言都有数组类型。
>     - **链表**操作基于指针，相对来讲前者简单。
> 
> > 两者各有优劣，通常较稳定的表选择顺序存储，频繁插入、删除操作的线性表选择链式存储。
